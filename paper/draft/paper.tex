\begin{abstract}
Fraud detection systems face challenges such as adversarial evasion, imbalanced data, and changing fraud patterns, where attackers make small modifications to bypass detection models. This paper presents a fraud detection system that improves robustness and adaptability by combining adversarial training, CTGAN-based data balancing, and a MAPE-K-based framework for continuous monitoring and automatic retraining. The system is designed to handle adversarial attacks, improve learning from imbalanced data, and adapt to evolving data distributions without manual intervention. Experimental results show that the proposed approach achieves promising performance with an F1 score of 0.9911 and ROC-AUC of 0.9966, while maintaining high robustness under adversarial perturbations and recovering performance under data drift. These results indicate that integrating robustness, data balancing, and autonomous adaptation provides an effective and practical solution for real-world fraud detection.

\keywords{Fraud Detection, MAPE-K Architecture, Adversarial Training, Data Drift, CTGAN, Autonomous Systems}
\end{abstract}

\section{Introduction}\label{sec:intro}

Fraud has become a major challenge in the digital economy, with global losses increasing due to the rise of online scams, phishing attacks, and social engineering schemes \cite{acfe2022report, javelin2023fraud}. These attacks often exploit both system weaknesses and human behavior, making fraud detection more difficult.

A key issue in modern fraud detection is that attackers constantly adapt their strategies to avoid being detected. Instead of making large changes, they introduce small and carefully designed modifications to transactions or behavior. These small changes can evade machine learning models, which are usually trained on static data. This reflects an adversarial setting, where inputs are intentionally modified to exploit model weaknesses and reduce detection accuracy.

Adversarial training has been proposed to improve model robustness against such attacks \cite{madry2018adversarial}. However, its use in fraud detection systems is still limited, especially in real-world applications.

Fraud detection systems also face challenges related to data. Real-world datasets are often imbalanced, with one class appearing much more frequently than the other. This can bias the model and reduce its ability to detect less frequent patterns \cite{dalpozzolo2015credit}. To address this, synthetic data generation methods such as CTGAN \cite{xu2019ctgan} can be used to improve class balance and support better learning.

In addition, fraud patterns change over time. Most existing models are static and need manual retraining when performance drops. This makes it difficult to keep up with new fraud strategies. Therefore, systems that can adapt automatically are important for maintaining performance over time.

To address these challenges, this paper presents an integrated fraud detection system that combines adversarial training, data balancing, and autonomous adaptation. Adversarial robustness is achieved through perturbation-based training, class imbalance is handled using CTGAN, and continuous adaptation is enabled through a MAPE-K-based control loop that monitors the system and triggers retraining when needed.

The proposed system provides a unified approach to handling adversarial behavior, imbalanced data, and evolving fraud patterns, enabling more reliable and adaptive fraud detection.

\section{Related Work}\label{sec:related}
Fraud detection has been widely studied in the literature. Traditional approaches use rule-based systems and basic machine learning models like logistic regression and decision trees. More recent work applies deep learning methods to detect fraudulent transactions. However, these approaches typically treat fraud detection as a static classification problem and do not address the challenges of changing fraud patterns.

Class imbalance is a well-known challenge in fraud detection. Dal Pozzolo et al. \cite{dalpozzolo2015credit} studied credit card fraud detection and showed that class imbalance significantly affects model performance. Various techniques have been proposed to handle this, including undersampling, oversampling, and synthetic data generation. CTGAN, introduced by Xu et al. \cite{xu2019ctgan}, generates synthetic tabular data that preserves the statistical properties of the original dataset. Several works that have applied CTGAN to fraud detection has shown promising results \cite{strelcenia2023synthetic}.

Data drift is another critical issue in fraud detection. As fraudsters change their tactics, the data distribution shifts over time. Goudar et al. \cite{goudar2026adaptive} proposed adaptive bagging strategies to address data drift in fraud detection systems. Various drift detection methods exist, including population stability index and Kolmogorov-Smirnov tests.

Adversarial robustness has gained attention in recent years. Madry et al. \cite{madry2018adversarial} showed that adversarial training can improve model robustness against evasion attacks. For tabular data, Simonetto et al. \cite{simonetto2024tabularbench} introduced TabularBench to benchmark adversarial robustness. Melo et al. \cite{melo2023adversarial} explored adversarial training specifically for tabular fraud detection data.

The MAPE-K architecture, introduced by IBM \cite{ibm2005autonomic}, provides a framework for building self-managing systems. Recent work has applied MAPE-K to various domains including microservices \cite{bucchiarone2022mape} and cloud systems \cite{oh2023mape}. However, limited work has explored MAPE-K for fraud detection specifically.

Our work differs from existing approaches by building an automated system that integrates data collection, augmentation, adversarial training, and autonomous adaptation using the MAPE-K architecture.

\section{Methodology}

This section describes our agentic fraud detection system built on the MAPE-K architecture with LLM-driven decision-making.

\subsection{MAPE-K Agent Architecture}

Our system implements MAPE-K (Monitor-Analyze-Plan-Execute using Knowledge) with specialized agents for autonomous fraud detection:

\begin{enumerate}
  \item \textbf{Monitor (M)}: DriftAgent + FeatureExtractionAgent
  \item \textbf{Analyze (A)}: RuleBasedScoringAgent + PatternLearningAgent  
  \item \textbf{Decision}: LLM Decision Agent (Ollama Llama 3)
  \item \textbf{Plan (P)}: TrainingAgent
  \item \textbf{Execute (E)}: BalanceAgent
  \item \textbf{Knowledge}: SQLite Knowledge Store
\end{enumerate}

System flow: Monitor $\rightarrow$ Analyze $\rightarrow$ Decision $\rightarrow$ Plan $\rightarrow$ Execute $\rightarrow$ Knowledge

\sububsection{Feature Extraction Agent}
Parses transaction data to extract:
\begin{itemize}
  \item Fraud labels: credential\_phishing, social\_authority\_scam, social\_urgency\_scam
  \item Key features: payment\_method, fraud\_channel, victim\_action
  \item Psychological tactics: urgency, fear, authority, reward
  \item Semantic profile: combined fraud signature
\end{itemize}

\sububsection{Rule-Based Scoring Agent}
Transparent weighted rules:
\begin{itemize}
  \item credential\_phishing: +0.25
  \item social\_authority\_scam: +0.25
  \item urgency: +0.15
  \item fear: +0.15
\end{itemize}
Score clamped 0-1.

\sububsection{Pattern Learning Agent}
Maps to patterns:
\begin{itemize}
  \item PHISHING: credential\_phishing
  \item SOCIAL\_ENGINEERING: authority + fear
  \item HYBRID: multiple labels
  \item Emerging: frequency $<$ 3
\end{itemize}

\sububsection{LLM Decision Agent (Ollama Llama 3)}
Replaces hardcoded rules with LLM reasoning. Given state (drift, metrics, KB), outputs:
\begin{itemize}
  \item Action: deploy / retrain\_L2 / retrain\_new\_strategy / stop
  \item Reason: contextual explanation
  \item Params: training parameters
\end{itemize}

\sububsection{Knowledge Store (SQLite)}
8 tables: transactions, predictions, reasoning, patterns, adversarial\_variants, execution\_logs, feedback, metrics.

\sububsection{Feedback Loops}
\begin{itemize}
  \item L1: Decision $\rightarrow$ Balance
  \item L2: Decision $\rightarrow$ Training  
  \item L3: Training $\rightarrow$ Monitor
  \item L4: Supervisor
  \item L5: Decision $\rightarrow$ Knowledge
\end{itemize}

\sububsection{Agentic Performance Metrics}
F1-score, PR-AUC, ROC-AUC, Adversarial Accuracy, FPR@Attack, Pattern Classification, Planning Accuracy, Human Override Rate, Robustness Gain.

\subsection{Data Collection and Annotation}
Collected from social media, LLM-based annotation for fraud labels.

\subsection{Data Balancing using CTGAN}
CTGAN generates synthetic minority samples for class balance.

\subsection{Adversarial Training}
FGSM for robustness:
\begin{equation}
x_{\text{adv}} = x + \epsilon \cdot \text{sign}(\nabla_x L(\theta, x, y))
\end{equation}
where $\epsilon = 0.05$. Trained on clean + adversarial samples. Evaluated at $\epsilon \in \{0.01, 0.05, 0.10, 0.20, 0.30\}$.

\section{Experimental Setup}\label{sec:experimental}

This section describes the experimental configuration used to evaluate the proposed fraud detection system, including dataset construction, model configuration, system architecture, evaluation protocol, and metrics.

\subsection{Dataset}

We used a dataset collected from social media platforms containing labeled fraud-related examples. The dataset consists of posts labeled as either fraudulent or normal (non-fraud), collected from Reddit fraud-focused communities over several months.

The raw dataset contained 9,000 posts. After LLM-based annotation and filtering of uncertain samples, 5,000 labeled examples were retained. For experimentation, a subset of 3,000 samples was used.

The dataset exhibits a skewed distribution with approximately 95\% fraud and 5\% non-fraud instances due to the nature of the data source. To address this imbalance, CTGAN is used to generate synthetic non-fraud samples during training.

For evaluation, the dataset is split into training and testing sets, ensuring that data used for model evaluation is not seen during training.

To simulate data drift, temporal variations are introduced in the test set by altering data distributions across time segments, allowing evaluation of the system’s adaptation capability.

\subsection{Model Configuration}

The base classifier used is XGBoost, a gradient boosting algorithm well-suited for tabular data. The model is configured with a learning rate of 0.1, maximum depth of 6, and 100 estimators. Binary cross-entropy is used as the objective function.

Adversarial training is incorporated using FGSM \cite{madry2018adversarial} with perturbation magnitude $\epsilon = 0.05$. The model is trained on both clean and adversarial examples to improve robustness.

CTGAN \cite{xu2019ctgan} is trained for 100 epochs with a batch size of 16. Generator and discriminator dimensions are set to 64 based on preliminary tuning.

Robustness evaluation is conducted using the TabularBench framework \cite{simonetto2024tabularbench}, which provides standardized adversarial evaluation for tabular models.

\subsection{System Architecture in Evaluation}

The experimental setup follows the system architecture illustrated in Figure~\ref{fig:architecture}. The system integrates data collection, augmentation, adversarial training, and adaptive monitoring within a unified pipeline.

During initial training, the pipeline processes collected data through annotation, balancing using CTGAN, and adversarial training before model deployment. Once deployed, the system operates in a continuous monitoring mode based on the MAPE-K framework \cite{ibm2005autonomic}.

Incoming data is analyzed for distributional changes using drift detection mechanisms, while model performance is continuously evaluated. When predefined thresholds are exceeded, the system triggers a retraining cycle that incorporates updated data and adversarial training.

This design enables controlled experimentation of robustness, drift adaptation, and data balancing within a single evaluation framework.

\subsection{Evaluation Protocol}

The evaluation is conducted across multiple dimensions to assess classification performance, robustness, and adaptability.

The model is first evaluated on clean test data to establish baseline performance. It is then evaluated under adversarial perturbations at varying strengths to measure robustness.

For drift evaluation, the system operates in a simulated deployment setting where incoming data distributions change over time. Drift detection mechanisms (PSI and KS) monitor these changes and trigger retraining when thresholds are exceeded.

The effectiveness of adaptation is measured by comparing model performance before and after retraining under drift conditions.

\subsection{Evaluation Metrics}

Performance is evaluated using standard classification metrics including accuracy, precision, recall, and F1 score. ROC-AUC is used to assess discrimination capability across thresholds.

For drift detection \cite{goudar2026adaptive}, Population Stability Index (PSI) and Kolmogorov-Smirnov (KS) tests are used. PSI values above 0.20 and KS values above 0.05 indicate significant drift.

Robustness is measured as the ratio of F1 score under adversarial perturbation to the clean F1 score.

Data quality of synthetic samples is evaluated using Jensen-Shannon Divergence (JSD), where values below 0.20 indicate strong similarity between real and generated distributions.

\subsection{Experimental Environment}

All experiments were conducted on a standard workstation using Python. Key libraries include scikit-learn for evaluation, XGBoost for modeling, and SDV for CTGAN implementation.

\section{Discussion}\label{sec:discussion}
This section discusses what our results mean and how they compare to existing work.

Our results show that combining CTGAN with adversarial training produces strong fraud detection performance. The CTGAN helps the model learn better by providing more balanced training data. Adversarial training then makes the model robust against attacks. Together, they achieve 98.34 percent robustness, meaning the model keeps working well even when someone tries to trick it.

The MAPE-K architecture adds real value by enabling automatic adaptation. Traditional fraud detection systems need humans to notice when something is wrong and manually retrain the model. Our system handles this automatically. When drift is detected, the system retrains itself and checks if performance improves. This reduces the need for human oversight and ensures the system stays effective as fraud patterns change.

One interesting finding is that the optimal amount of synthetic data is around 700 samples. Adding more did not help and sometimes hurt performance. This suggests that while synthetic data is useful, too much can introduce noise. Finding the right balance is important.

Our system performed well even under data drift. When fraud patterns changed, the model could still maintain F1 above 0.90 after automatic retraining. This is better than many static systems that would fail completely under the same conditions.

There are some limitations to consider. First, we collected data from social media which may not represent all types of fraud. Some fraud schemes may not be discussed publicly online. Second, we used XGBoost which is a traditional machine learning model. More complex deep learning approaches might achieve even better results. Third, while we use FGSM-based adversarial training, more sophisticated attacks like PGD could be explored in future work to further test robustness.

The system demonstrates strong performance metrics that suggest potential for real-world application. The F1 score of 0.9911 and robustness of 98.34 percent are encouraging, though further validation would be needed for production deployment. The automatic adaptation means the system can handle changing fraud patterns without constant human attention.

Compared to related work, our system integrates multiple components that others typically handle separately. Most papers address class imbalance or adversarial training in isolation. Our complete pipeline from data collection to autonomous adaptation integrates these components in a unified framework.

Future work could explore different model architectures, test on more diverse datasets, and implement more advanced adversarial training methods. The MAPE-K framework could also be extended with more sophisticated planning algorithms.

Overall, our automated fraud detection system demonstrates how combining established techniques like CTGAN and adversarial training with the MAPE-K architecture creates a robust and adaptive solution for real-world fraud detection challenges.

\section{Conclusion}\label{sec:conclusion}
This paper presented an automated fraud detection system built on the MAPE-K architecture. The system integrates five specialized agents (Ingestion, Drift, Evaluation, Training, Balance) following the M $\rightarrow$ A $\rightarrow$ P $\rightarrow$E order, with feedback loops and Supervisor oversight.

Our experiments showed promising results. The system achieves F1 score of 0.9911 and ROC-AUC of 0.9966 on real fraud data. The robustness against adversarial attacks is 98.34 percent, demonstrating that the model can handle attempts to evade detection. The MAPE-K architecture successfully detects data drift and triggers automatic retraining, achieving F1 of 0.92 even under changing fraud patterns.

The key contributions of this work are a complete automated fraud detection pipeline that handles data collection, augmentation, adversarial training, and autonomous adaptation, a practical implementation of MAPE-K for fraud detection that demonstrates measurable learning adaptation, evidence that combining CTGAN with adversarial training produces robust models, and strong experimental results validated on real-world data.

For practitioners, this system offers a deployable solution that reduces manual oversight through automatic adaptation. For researchers, the integration of multiple techniques into a coherent framework provides a foundation for further work.

Future directions include exploring more advanced model architectures, testing on additional datasets from different sources, and implementing more sophisticated adversarial training methods. The MAPE-K framework could also be extended with better planning algorithms to optimize retraining decisions.

In conclusion, our automated fraud detection system demonstrates how automated modules can work together to handle the complex challenges of modern fraud detection. By combining established techniques with autonomous adaptation, we create a robust and practical solution for real-world fraud detection needs.
