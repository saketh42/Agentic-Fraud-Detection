\begin{abstract}
This paper presents an automated fraud detection system built on the MAPE-K architecture with LLM-assisted decision-making. The system uses XGBoost as the base classifier, with CTGAN for class imbalance and FGSM adversarial training for robustness. The MAPE-K loop monitors data drift and triggers automatic retraining. A local LLM (Ollama Llama 3) assists by providing contextual reasoning over historical performance data, augmenting threshold-based decision rules. Experimental results demonstrate F1 score of 0.9877, ROC-AUC of 0.9869, 98.34% robustness under adversarial perturbations, and successful drift recovery (F1: 0.9726 $\rightarrow$ 0.9929 after retraining).

\keywords{Fraud Detection, MAPE-K Architecture, LLM Decision Support, Adversarial Training, Data Drift, CTGAN}
\end{abstract}

\section{Introduction}\label{sec:intro}

Fraud has become a major challenge in the digital economy, with global losses increasing due to online scams, phishing attacks, and social engineering schemes \cite{acfe2022report, javelin2023fraud}. Modern fraudsters adapt constantly to evade detection, making traditional static models insufficient.

This paper presents an automated fraud detection system combining: (1) XGBoost classifier with CTGAN for class imbalance; (2) FGSM adversarial training for robustness; (3) MAPE-K loop for drift detection and automatic retraining; (4) Local LLM (Ollama Llama 3) for contextual decision support, augmenting threshold-based rules with historical performance context.

Key contributions: (1) Integrated pipeline handling data balancing, adversarial training, and automatic adaptation; (2) MAPE-K based drift detection with automatic retraining; (3) LLM-assisted decision-making using local deployment for privacy; (4) Experimental evaluation demonstrating F1 of 0.9877, 98.34% robustness, and drift recovery.

\section{Related Work}\label{sec:related}
Fraud detection has been widely studied using traditional ML and deep learning methods. Class imbalance is addressed using CTGAN \cite{xu2019ctgan} for synthetic data generation. Data drift detection uses PSI and KS tests \cite{goudar2026adaptive}. Adversarial training \cite{madry2018adversarial} improves robustness against evasion.

The MAPE-K architecture \cite{ibm2005autonomic} enables self-managing systems. Recent work applies MAPE-K to microservices \cite{bucchiarone2022mape} and cloud systems \cite{oh2023mape}. 

Our work integrates XGBoost, CTGAN, FGSM adversarial training, and MAPE-K loop with LLM-assisted decision-making in a unified pipeline.

\subsection{Data Balancing (CTGAN)}
Real-world fraud data is highly imbalanced. CTGAN \cite{xu2019ctgan} generates synthetic minority samples to balance training data.

\subsection{Adversarial Training}
Fraudsters manipulate inputs to evade detection. FGSM adversarial training improves robustness:
\begin{equation}
x_{\text{adv}} = x + \epsilon \cdot \text{sign}(\nabla_x L(\theta, x, y))
\end{equation}
where $\epsilon = 0.05$. The model is trained on clean + adversarial samples.

\subsection{MAPE-K Control Loop}
The MAPE-K loop enables automatic adaptation. Monitor detects drift using PSI and KS tests. Analyze evaluates model performance. Decision determines whether to retrain. Plan executes retraining with updated data. Execute deploys the retrained model.

\sububsection{LLM-Assisted Decision-Making}
A local LLM (Ollama Llama 3) assists by providing contextual analysis. We evaluate the LLM decisions against rule-based baseline.

\subsubsubsection{Prompt Design}

\end{verbatim}

\section{Experimental Setup}\label{sec:experimental}

This section describes the experimental configuration used to evaluate the proposed fraud detection system, including dataset construction, model configuration, system architecture, evaluation protocol, and metrics.

\subsection{Dataset}

We used a dataset collected from social media platforms containing labeled fraud-related examples. The dataset consists of posts labeled as either fraudulent or normal (non-fraud), collected from Reddit fraud-focused communities over several months.

The raw dataset contained 9,000 posts. After LLM-based annotation and filtering of uncertain samples, 5,000 labeled examples were retained. For experimentation, a subset of 3,000 samples was used.

The dataset exhibits a skewed distribution with approximately 95\% fraud and 5\% non-fraud instances due to the nature of the data source. To address this imbalance, CTGAN is used to generate synthetic non-fraud samples during training.

For evaluation, the dataset is split into training and testing sets, ensuring that data used for model evaluation is not seen during training.

To simulate data drift, temporal variations are introduced in the test set by altering data distributions across time segments, allowing evaluation of the system’s adaptation capability.

\subsection{Model Configuration}

The base classifier used is XGBoost, a gradient boosting algorithm well-suited for tabular data. The model is configured with a learning rate of 0.1, maximum depth of 6, and 100 estimators. Binary cross-entropy is used as the objective function.

Adversarial training is incorporated using FGSM \cite{madry2018adversarial} with perturbation magnitude $\epsilon = 0.05$. The model is trained on both clean and adversarial examples to improve robustness.

CTGAN \cite{xu2019ctgan} is trained for 100 epochs with a batch size of 16. Generator and discriminator dimensions are set to 64 based on preliminary tuning.

Robustness evaluation is conducted using the TabularBench framework \cite{simonetto2024tabularbench}, which provides standardized adversarial evaluation for tabular models.

\subsection{System Architecture in Evaluation}

The pipeline integrates: (1) CTGAN for data balancing; (2) XGBoost classifier; (3) FGSM adversarial training; (4) MAPE-K monitoring loop; (5) LLM-assisted decision-making.

\subsection{Evaluation Protocol}

The evaluation is conducted across multiple dimensions to assess classification performance, robustness, and adaptability.

The model is first evaluated on clean test data to establish baseline performance. It is then evaluated under adversarial perturbations at varying strengths to measure robustness.

For drift evaluation, the system operates in a simulated deployment setting where incoming data distributions change over time. Drift detection mechanisms (PSI and KS) monitor these changes and trigger retraining when thresholds are exceeded.

The effectiveness of adaptation is measured by comparing model performance before and after retraining under drift conditions.

\subsection{Evaluation Metrics}

Performance is evaluated using standard classification metrics including accuracy, precision, recall, and F1 score. ROC-AUC is used to assess discrimination capability across thresholds.

For drift detection \cite{goudar2026adaptive}, Population Stability Index (PSI) and Kolmogorov-Smirnov (KS) tests are used. PSI values above 0.20 and KS values above 0.05 indicate significant drift.

Robustness is measured as the ratio of F1 score under adversarial perturbation to the clean F1 score.

Data quality of synthetic samples is evaluated using Jensen-Shannon Divergence (JSD), where values below 0.20 indicate strong similarity between real and generated distributions.

\subsection{Experimental Environment}

All experiments were conducted on a standard workstation using Python. Key libraries include scikit-learn for evaluation, XGBoost for modeling, and SDV for CTGAN implementation.

\section{Results}\label{sec:results}
This section presents experimental results for our automated fraud detection system.

\subsection{Classification Performance}
Our system achieves strong results on the fraud detection task:
\begin{itemize}
  \item F1-Score: 0.9877
  \item ROC-AUC: 0.9869  
  \item PR-AUC: 0.9500
  \item Precision: 0.99
  \item Recall: 0.99
  \item Robustness: 98.34\%
\end{itemize}

These results demonstrate that combining CTGAN data augmentation with adversarial training produces an effective fraud detector.

\subsection{Drift Detection and Adaptation}
We tested drift detection and adaptation using controlled experiments:

\begin{table}[h]
\centering
\caption{Drift Adaptation Results}
\begin{tabular}{|l|c|c|c|}
\hline
Phase & F1-Score & ROC-AUC & Drift Detected \\
\hline
Initial (clean) & 0.9877 & 0.9869 & No \\
After Drift & 0.9726 & 0.9271 & Yes \\
After Retrain & 0.9929 & 0.9943 & No \\
\hline

Drift was successfully detected with PSI = 8.09 (threshold: 0.20) and KS = 0.50 (threshold: 0.05). The ROC-AUC dropped from 0.9869 to 0.9271 (6.0\% degradation) under drift. After retraining on drifted data, the system recovered to F1 = 0.9929.

This demonstrates the practical value of our MAPE-K architecture. The system detects drift and triggers automatic retraining.

\subsection{Robustness under Adversarial Attacks}
We tested adversarial robustness with perturbation levels from 0.01 to 0.30. The model maintained F1 > 0.97 even at highest perturbation, demonstrating effective adversarial training.

\subsection{Pattern Learning}
The Pattern Learning Agent achieved 100\% accuracy in classifying fraud patterns (PHISHING, SOCIAL\_ENGINEERING, HYBRID).

The combination of all components produces good results for automated fraud detection.

\section{Discussion}\label{sec:discussion}
This section discusses what our results mean and how they compare to existing work.

Our results show that combining CTGAN with adversarial training produces strong fraud detection performance. The CTGAN helps the model learn better by providing more balanced training data. Adversarial training then makes the model robust against attacks. Together, they achieve 98.34 percent robustness.

The MAPE-K architecture adds value by enabling automatic adaptation. Traditional fraud detection systems need humans to notice when something is wrong and manually retrain. Our system handles this automatically. The LLM provides contextual analysis to augment threshold-based decisions.

One finding is that the optimal amount of synthetic data is around 700 samples. Adding more did not help and sometimes hurt performance.

Our system performed well under data drift. When fraud patterns changed, the model maintained F1 above 0.90 after automatic retraining.

Limitations: First, we collected data from social media which may not represent all fraud types. Second, we used XGBoost. More complex models might achieve better results. Third, FGSM adversarial training could be extended to PGD.

Overall, our automated fraud detection system demonstrates how combining CTGAN and adversarial training with MAPE-K creates a robust and adaptive solution.

\section{Conclusion}\label{sec:conclusion}
This paper presented an automated fraud detection system built on the MAPE-K architecture with LLM-assisted decision-making.

Our experiments showed promising results. The system achieves F1 score of 0.9877 and ROC-AUC of 0.9869. The robustness against adversarial attacks is 98.34 percent. The MAPE-K loop detects data drift and triggers automatic retraining, achieving F1 of 0.9929 after recovery (from 0.9726).

Key contributions: (1) Integrated pipeline with CTGAN, adversarial training, and MAPE-K loop; (2) LLM-assisted decision-making using local deployment; (3) Experimental evaluation demonstrating drift detection and recovery.

Future directions include exploring different model architectures, testing on more diverse datasets, and implementing more advanced adversarial training.

In conclusion, our automated fraud detection system demonstrates how established techniques combined with MAPE-K create a robust and practical solution for real-world fraud detection.
